微信内置浏览器 android与ios的区别

因为android开源，所以对第三方app的webview没太严格的要求，你可以使用自己的内核，微信使用x5内核，其实也是基于webkit内核封装的，所以你公众号里阅读文章的时候往下拉页面才会看见QQ浏览器x5内核提供技术支持。

但是ios是有严格的限制条件，你可以试下，相同的文章你在ios的微信里看见的是网页由mp.weixin.qq.com提供，这说明ios使用的不是微信自带的x5内核，原因就是ios限制了除safari外的任何浏览器内核存在，所以ios微信就是直接使用的普通webview，完全由ios提供技术支持。

一体化设计方法与实践

我们借鉴了建筑的工程化思维模式，构建了一套设计协作的方法，我们称之为“一体化设计”。这套方法通过将感性的视觉设计思维：颜色、字体、间距等设计规则化并封装，形成基本的设计元素；通过将交互设计对于网站信息架构的理解，将网站页面分解为若干交互层级，形成设计模式；再将常用产品功能穷举并设计组件；最终将这些元素，框架，组件封装为工具库以插件的形式介入到设计软件中以服务于设计师本人，重定义设计流程，将各个设计师的专业技能模式化，收入到工具中反哺给其他设计师。

通过一体化设计方法，我们可以使设计效率翻倍，让初级设计师迅速具备高级设计师的部分设计技能。让设计师从繁杂的设计中释放出更多精力，用于专业研究、用于了解用户，用于设计创新。

通过一体化的设计方式，可将产品经理、设计师、前端工程师通过结构化的协作方式、通用的设计语言有效串联，大大提升产品设计开发效率，为让试错成本进一步降低。

10个案例说明什么是产品模型

产品模型=商业模式+产品架构+运营体系。

商业模式指产品的市场潜力，综合了用户与营收价值。
产品架构指产品设计上的框架与核心系统。
运营体系与产品架构类似，指运营上的组织流程与重点难点。

–文／纯银V（简书作者） 著作权归作者所有

10个案例说明什么是产品模型

在沟通中，我们要避免自以为是的情况，自以为自己讲清楚了，自以为对方能明白自己的意思。

我的工作需要和各部门打交道，我一开始发现，我告诉别人要做什么，最后的结果，往往令人失望。这时，我才发现问题出在沟通上面，比如我让设计师设计东西，给她描述了我想要的东西，她一脸茫然，问我你到底想要什么呢？

我说我就想要温馨的感觉，你按你的理解先设计就可以。这样的结果，我肯定不满意，再让设计师修改，额外增加她很多工作量。

我发现这一问题后，再沟通时，我会找一些想要的感觉的图片给她，她就知道找一个什么样的图片，文字部分，我也提前给她准备好。在布置任务时，我详细解释，并且和她确认，这样就不会出现无效沟通导致工作效率降低，大家都不开心的局面了。

做A事情的时候，就是A事情，而不要想着B事情。无论参加聚会也好，集体活动也罢，都全身心投入，不想其它的事情，我发现每次都收获很多。当我们全身心投入一件事情时，我们会变得快乐，并且效率很高。
